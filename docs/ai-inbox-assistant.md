# LifeMap AI Inbox Assistant Logic

Дата: 2026-06-25

Этот документ фиксирует логику будущего AI-помощника внутри LifeMap. Сейчас помощник ещё не подключён к модели, но UI и правила уже заложены, чтобы позже подключить реальный мозг без пересборки концепции.

## Роль AI Inbox

AI Inbox — это не список задач. Это входящий фильтр для постов, ссылок, промптов, инструментов, идей, новостей и наблюдений из Telegram и других источников.

Каждый входящий сигнал должен пройти разбор:

1. Что это такое?
2. К какой теме или проекту относится?
3. Можно ли применить это в LifeMap, Sleda.net, AI Inbox, обучении вайб-кодингу или другом проекте?
4. Это нужно сохранить, превратить в задачу, вынести в библиотеку или отправить в архив?

## Обязательная структура разбора сигнала

Для каждого сигнала помощник должен выделять:

- короткий заголовок;
- суть материала;
- категорию;
- платформы/инструменты;
- тип полезного актива;
- возможное применение;
- следующий шаг;
- уровень приоритета;
- решение: разобрать, сохранить в библиотеку, создать задачу, применить в проекте, исследовать или архивировать.

## Категории

Основные категории:

- Prompt — промпты, инструкции для моделей, CLAUDE.md, review.md, системные подсказки;
- AI Tool — сервисы, приложения, MCP-коннекторы, библиотеки, агенты;
- Workflow — повторяемые процессы, пайплайны, автоматизации;
- Code/Codex — Codex, GitHub, Claude Code, Cursor, opencode, разработческие практики;
- Design/UX — дизайн, интерфейсы, UI, UX, дизайн-системы;
- Research/News — новости рынка, технологии, RAG, scraping, ML/RAG-методы;
- Business/Monetization — деньги, клиенты, оплата, лимиты, бизнес-модель;
- Security/Legal — безопасность, приватность, юридические ограничения;
- Idea — продуктовая или контентная идея;
- Reference — справочный материал без немедленного действия.

## Типы активов

Один пост может дать несколько активов:

- Prompt;
- Workflow;
- Tool link;
- Idea;
- Instruction;
- Task;
- Source.

Например, пост про Claude Code может одновременно быть Tool link, Workflow и Instruction.

## Правила помощника

1. Не превращать сигнал в задачу автоматически без решения пользователя.
2. Не смешивать входящие сигналы с рабочими задачами LifeMap.
3. Если в посте есть промпты — предложить вынести их в библиотеку промптов по цели применения.
4. Если в посте есть инструмент — сохранить его в библиотеку инструментов с описанием: что делает, где применить, какие ограничения.
5. Если материал требует проверки — помечать как Research, а не принимать на веру.
6. Если материал связан с безопасностью, обходами, лимитами, приватностью или деньгами — добавлять предупреждение о проверке и осторожности.
7. Если сигнал применим к текущему проекту — предлагать следующий маленький шаг, а не большой размытый план.

## Будущий сценарий в UI

Пользователь присылает пост Telegram-боту. LifeMap сохраняет сигнал в Notion и показывает его в AI Inbox.

Дальше пользователь открывает сигнал и может выбрать:

- Разобрано;
- Архив;
- В библиотеку промптов;
- В библиотеку инструментов;
- Создать задачу;
- Привязать к проекту;
- Спросить помощника.

Помощник должен видеть текущую ветку карты, текущий фокус, источник сигнала и связанные проекты.
